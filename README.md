# Fitness App Workout Dataset — End-to-End EDA

An exploratory data-analysis (EDA) project that cleans, merges, and analyzes a fitness-app dataset of **~300 members** and **~4,100 workout sessions** to surface behavioral patterns in gym engagement — built as a single, reproducible Jupyter notebook.

> This is a **data-analysis / EDA** project (cleaning → feature engineering → merge → descriptive analysis → visualization). It is not a machine-learning / predictive-modeling project.

## Problem / Questions

The dataset simulates a gym's member and workout records. The analysis answers a set of practical business questions a fitness operator would care about:

- **Is engagement growing?** How does the number of workouts trend over time?
- **Which members are most active?** Do membership tiers (Basic / Standard / Premium) differ in workouts completed per member?
- **What drives calorie burn?** How do average calories burned compare across workout types (Cardio, Strength, Yoga, Class)?
- **Does effort scale with time?** What is the relationship between workout duration and calories burned?
- **When do people train?** How are workouts distributed across the hours of the day, and does this differ by membership tier?
- **Who are the members?** What does the member BMI distribution look like, and are there differences in calorie burn by gender?

## Data

The project follows an explicit **raw → clean → merged** pipeline. Both source tables are cleaned independently, enriched with engineered features, then joined into a single analysis-ready table.

| Stage | Files | Notes |
|-------|-------|-------|
| Raw | `members_raw.csv`, `workouts_raw.csv` | ~300 members, ~4,100 workout sessions as originally recorded |
| Clean | `members_clean.csv`, `workouts_clean.csv` | Standardized categories, imputed missing values, engineered features |
| Merged | `fitness_merged_clean.csv` | Workouts left-joined to members on `member_id` — the analysis-ready table |

**Cleaning & feature engineering performed in the notebook:**

- **Members** — normalized `gender` (Male / Female / Other) and `city` (title case); imputed missing `age` with the overall median and missing `height`/`weight` with per-gender medians; derived `BMI` from height and weight; binned members into `age_group` buckets (18–25, 26–35, 36–45, 46–60).
- **Workouts** — normalized `workout_type` into clean categories (Cardio, Strength, Yoga, Class, Other), including fixing a "strenght" spelling variant; treated out-of-range values as missing and imputed by workout-type median (`duration_min` outside 10–180, non-positive `calories_burned`, `satisfaction_rating` outside 1–5, `avg_heart_rate` outside 60–200); engineered time features (`workout_month`, `day_of_week`, `hour_of_day`) and `calories_per_min`.
- **Merged** — added `tenure_days` (days between `join_date` and workout); dropped rows still missing critical fields after imputation.

A full column-level reference — table, dtype, and description for every field — is committed as [`data_dictionary.csv`](data_dictionary.csv) and is also regenerated by the notebook.

> **Note on data provenance:** the dataset is synthetic / programmatically generated. Satisfaction ratings and heart-rate values are simulated, so findings illustrate analytical workflow rather than real-world member behavior.

## Key Findings

Drawn from the notebook's analysis and captured in [`summary.md`](summary.md):

1. **Rising engagement over time.** Monthly workout counts show an overall upward trend, indicating growing member engagement and/or an expanding membership base.
2. **Premium members are the most active.** Premium members complete more workouts per person on average than Basic and Standard members.
3. **Cardio burns the most calories.** Cardio sessions have the highest average calories burned per session; Yoga and other low-intensity activities burn the least — consistent with workout intensity.
4. **Duration strongly predicts calorie burn.** There is a clear positive relationship between workout duration and calories burned, with variation increasing for longer sessions.
5. **Evening workouts dominate.** Most sessions occur in the late afternoon and evening; Premium members show relatively more off-peak daytime workouts than other tiers.
6. **Member BMI skews toward overweight/obese.** The BMI distribution suggests many members start in the overweight to moderately obese range — consistent with a gym population aiming to lose weight and improve fitness.

## Running Locally

```bash
# 1. Clone the repository
git clone https://github.com/taha-yasin-saad/Fitness-App-Workout-Dataset.git
cd Fitness-App-Workout-Dataset

# 2. (Recommended) create a virtual environment
python -m venv .venv
# Windows:  .venv\Scripts\activate
# macOS/Linux:  source .venv/bin/activate

# 3. Install dependencies
pip install pandas numpy matplotlib jupyter

# 4. Launch the notebook
jupyter notebook fitness_eda_project.ipynb
```

With `members_raw.csv` and `workouts_raw.csv` present in the project root, running all cells reproduces the cleaning steps, regenerates the cleaned/merged CSVs and the data dictionary, and renders every visualization.
